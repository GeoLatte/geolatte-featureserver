This API exposes a generic storage and query service for geojson features. The features are organised into databases and collections (as in MongoDB).

Currently two persistence options are provided: Postgresql and MongoDB.

/databases/{database}/{collection} get put delete

a collection (table) of GeoJSON features.

get

Returns the metadata associated with this collection. The metadata consists of:

  • the name of the collection
  • the spatial extent of the complete collection
  • the index-level (only used for MongoDB)
  • the type of the (required) id property (either 'text' or 'decimal')
  • the number of features currrently in the collection

delete

/databases/{database}/{collection}/tx/update post

post

POST an update specification. An update specification is a json object having query and update properties. The query property holds a query expression that specifies the features to update. The update property holds

  • in case of MongoDB, either a replacement feature (document), or an object consisting of only update operators. (The update will be performed with multi=true);
  • in case of Postgresql, a replacement feature.

/databases/{database}/{collection}/query get

get

Executes a query on the specified collection and returns a stream of features (separated by a newline character). The bbox and query parameters are ANDed together.

get /databases/{database}/{collection}/query

Executes a query on the specified collection and returns a stream of features (separated by a newline character). The bbox and query parameters are ANDed together.

  • Request
  • Response

URI Parameters

  • database: required(string)
  • collection: required(string)

Query Parameters

  • bbox: (string)

    Bounding box in string-representation (<min-x>,<min-y>,<max-x>,<max-y>)

    Example:

    0,0,90,90
  • query: (string)

    Query expression

    Example:

    Properties.foo = 'bar'
  • start: (integer)

    Skips the specified number of features before returning features

  • limit: (integer)

    Specifies the maximum number of features to return

  • projection: (string)

    specifies a comma-separated list of feature property paths. If this parameter is present, the output contains all and only the specified property paths. If the input Json doesn't contain a specified property path, it will be add in the output with a value of null.

    Example:

    properties.foo,properties.bar,properties.baz
  • sort: (string)

    specifies a comma-separated list of feature property paths. If this parameter is present, the output is sorted on the specified property paths (according to the order specified in the 'sort-direction'-parameter).

    Example:

    properties.foo,properties.bar,properties.baz
  • sort-direction: (string)

    specifies a comma-separated list of sort directions (either 'asc' or 'desc') to use when sorting the corresponding property paths specified in the sort parameter. If this parameter is absent, or does not specify a direction for a sort property, the ascending sort order will be used by default.

  • with-view: (string)

    The name of the ViewDef to apply. A ViewDef has query and projection properties. The ViewDef query expression will be conjoined (AND-ed) to the query request parameter, and the ViewDef projection property will be concatenated to the projection request parameter.

  • sep: (string)

    The separator to use in the CSV result this parameter is ignored when the accepted media type is other than CSV

  • fmt: (string)

    Either "csv" or "json". Hack for specifying the Accepted media type for the response, when no Accept header can be set by the client.

  • filename: (string)

    Hack to add a filename to content-disposition attachment header field to force browser to download the data with given filename.

HTTP status code 200

Successfully executed the query

Body

Media type: application/vnd.geolatte-featureserver+json;version=1.0

Type: any

Example:

{id = 433, geometry":{"type":"Point","crs":{"properties":{"name":"EPSG:31370"},"type":"name"},"bbox":[158000,191400,158050,191400],"coordinates":[158000,191400]},"type":"Feature","properties":{"some-property":"some-value"}}
{id = 434, geometry":{"type":"Point","crs":{"properties":{"name":"EPSG:31370"},"type":"name"},"bbox":[158001,191401,158051,191401],"coordinates":[158001,191401]},"type":"Feature","properties":{"some-property":"some-other-value"}}
....

Media type: application/vnd.geolatte-featureserver+csv;version=1.0

Type: any

Example:

id, geometry-wkt, some-property
433,SRID=31370;POINT(158000 191400),"some-value"
434,SRID=31370;POINT(158001 191401),"some-other-value"

/databases/{database}/{collection}/indexes/{indexName} put get delete

put

Creates the index on the specified collection.

If the regex property is specified and set to true a GIST index will be created using the tigram matching functions of the pg_trgm extension. If this extension has not been created in the database, index creation will fail.

delete

Deletes the specified index
